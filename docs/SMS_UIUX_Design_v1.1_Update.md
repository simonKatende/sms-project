**1. Scope of This Update (v1.1)**

This document is a targeted update to the UI/UX Design Document v1.0. Only screens and components that have changed or been added are documented here. All design system principles, colour palette, typography, component library, navigation structure, and responsive guidelines from v1.0 remain in force.

  ----------------------------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **Change Area**                     **What Changed**
  **Pupil registration form**         Added pupil photo upload field. Added separate WhatsApp number field. Added bursary section with scheme selector, standard fees display, agreed net fees input, and auto-computed discount.
  **Fees module --- Bulk billing**    New screen: Bulk Bill Generation. Allows Bursar to generate bills for an entire class in one action.
  **Fees module --- Fee statement**   New action on pupil bill page: Generate Fee Statement PDF for report envelope.
  **Academics --- Score entry**       Clarified NULL vs zero distinction. Added F9 indicator highlighting. Updated grade labels to 9-point scale.
  **Academics --- Report cards**      Updated to reflect DOS-only generation. Added report card preview matching school sample format (photo, SCH PAY NO., BOT/MOT/EOT columns, teacher initials, grade guide, requirements, next-term dates).
  **Terminology**                     \'Student\' replaced with \'Pupil\' on all screens, labels, and form fields.
  ----------------------------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

  ------------------------------ --------------------------------------------------------------------------------- ---------------------------------------------------------------------
  **Element**                    **Behaviour**                                                                     **Notes**
  **Photo upload**               Multer accepts JPEG/PNG ≤2MB. Sharp resizes to 300×400px on server.               If no photo at registration, can be added later from pupil profile.
  **Standard fee (read-only)**   Auto-populated when class and section are selected. Reads from fee\_structures.   Updates if class/section changes before save.
  **Bursary checkbox**           Reveals/hides bursary section. Unchecked = no bursary, standard billing.          
  **Agreed net fees input**      Required when bursary checkbox is ticked. Must be less than standard fee.         Validation: net fees \> 0 and \< standard\_fee.
  **Auto-computed discount**     = standard\_fee - agreed\_net\_fees. Displayed in real-time as user types.        Read-only. Stored as discount\_ugx in pupil\_bursaries.

**3. New Screen: Bulk Bill Generation**

This new screen allows the Bursar to generate term bills for an entire class in a single action, rather than per pupil. The system uses each pupil\'s class, section, and bursary record to compute bills automatically.
